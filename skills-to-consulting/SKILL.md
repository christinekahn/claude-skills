# Skills → Consulting Skill

A framework for productizing expertise: taking one thing you're genuinely good at and
turning it into a bookable, priced offer — without inventing a fake case study or
disclosing anything proprietary from a past employer.

Use this skill any time someone says things like: "how do I sell this," "turn my skill
into a service," "what should I charge for this," "help me write a pitch for X," or
"build a skills/services page."

---

## Workflow Overview

1. **Identify the skill** — name it, and generalize it away from any one employer or
   client so it reads as a transferable pattern, not proprietary IP.
2. **Build the three-tier engagement ladder** — Working Session, Applied Sprint,
   Fractional Partnership.
3. **Write the one-page pitch** — headline, description, "where it fits," and a CTA.
4. **Prep scoping questions** — what to ask a prospect before quoting anything.
5. **Draft outreach copy** — a short message to open the conversation.

If the user says "do all of it for [skill name]," produce all five in one pass.

---

## 1. Identify and Generalize the Skill

Before writing anything, ask (or infer from context):

- What's the skill, in one sentence? ("I can redesign a disaster-recovery setup so
  failover is faster and tested regularly.")
- Where did this get sharp? (a past role, a project, a hobby) — capture this for your
  own confidence, but do NOT put employer names, client names, or exact proprietary
  numbers into the public-facing pitch.
- What's the generic version of the outcome? Convert specifics like "$11M saved at
  [Company]" into a use-case description like "cuts cost-to-serve in high-volume
  support queues." The specific dollar figure can live on a resume/LinkedIn in the
  context of "results I've delivered" — it should NOT be reused as the selling point
  for a packaged, resellable version of the same pattern. That crosses from "here's
  proof I'm good" into "here's your company's internal system for sale" — a line to
  stay well clear of.

**Red flags to strip out of any public pitch:**
- Named employers or clients paired with a specific system description
- Screenshots, internal tool names, or verbatim internal process names
- Metrics that are unique enough to be traceable to one specific, identifiable program

**Keep instead:**
- The pattern / method itself, described generically
- The type of environment it suits ("high-volume support," "regulated fintech,"
  "multi-team programs without a single owner")
- Aggregate, resume-level credibility ("12+ years across fintech, media, and retail
  tech") kept separate from the specific pitch

---

## 2. Build the Three-Tier Engagement Ladder

Every packaged skill should scale from "barely a commitment" to "ongoing partnership."
Use this ladder as the default structure unless the user has a reason to deviate:

### Tier 1 — Working Session
- **Format:** A single half-day or full-day session.
- **Purpose:** Prove the fit. Apply the skill/framework to one real problem the
  prospect already has, live, in the room.
- **Deliverable:** A concrete artifact from the session (a plan, a diagram, a working
  first draft) — not just a conversation.
- **Positioning:** The lead-magnet tier. Low friction, low price, easy yes.

### Tier 2 — Applied Sprint
- **Format:** 2–4 weeks, embedded.
- **Purpose:** Apply the pattern to a live initiative, not a hypothetical — with the
  prospect's own team, systems, and constraints.
- **Deliverable:** The pattern implemented (or clearly scoped for implementation),
  plus documentation so the team can maintain it after the sprint ends.
- **Positioning:** The "let's actually do this" tier — proof of value at real stakes.

### Tier 3 — Fractional Partnership
- **Format:** Ongoing, typically 1–3 days a week.
- **Purpose:** Embedded leadership across a broader roadmap, not just one pattern —
  the relationship graduates from "apply this skill" to "help me run this."
- **Deliverable:** Whatever cadence fits (weekly syncs, governance artifacts, roadmap
  reviews) — defined jointly in the first two weeks.
- **Positioning:** The retention tier. Priced and scoped like a part-time hire, not a
  project.

Ask the user for their own naming/branding preferences, but default to descriptive,
literal tier names over cute ones — clarity converts better than cleverness here.

## 3. Write the One-Page Pitch

For each skill being packaged, produce:

```
[Tag/category]
### [Skill name — plain, descriptive, no jargon for jargon's sake]

[2–3 sentence description: what the pattern does, and the situation it solves —
written so a stranger with no context understands it immediately.]

Where it fits: [one line — the type of team/problem/environment this suits]

[CTA verb phrase] →
```

**Headline rules:**
- Name the skill by what it *does*, not by an internal codename.
- Avoid superlatives ("best-in-class," "revolutionary") — let the specificity of the
  description carry the credibility instead.

**CTA rules:**
- Always end with an action verb aimed at the reader's problem, not at the seller's
  product: "Build this for your product," "Assess your resiliency," "Train your team"
  — not "Learn more" or "Contact us."

---

## 4. Scoping Questions (ask before quoting anything)

- What does this problem cost you today — in time, money, or risk — if it stays
  unsolved for another quarter?
- Who else needs to be involved for this to stick after the engagement ends?
- What's already been tried, and why didn't it work?
- What would make this a clear, obvious win in your eyes by the end of a Working
  Session?
- Is there a hard deadline (compliance, launch, budget cycle) shaping the timeline?

Use the answers to route the prospect to the right tier — don't default everyone to
the highest tier just because it's the most profitable one. A well-scoped Working
Session that converts to a Sprint later is worth more than an oversold Partnership
that churns in month two.

---

## 5. Outreach Copy Template

Keep it short, specific to their situation, and low-pressure:

```
Hi [Name] — [one sentence referencing something specific and true about their
situation, not a generic compliment].

I've spent [N years / a career] doing [the generalized skill], most recently
[one generic descriptor of scale/domain, no employer name]. If [specific problem
you noticed or they mentioned] is on your plate, I run short working sessions to
get a real first pass on it — no deck, just working the problem together.

Worth a short conversation?

[Name]
```

Adjust tone to match the channel (LinkedIn DM vs. email vs. warm intro), but keep
the core shape: specific observation → credibility in one line → concrete, low-
commitment ask.

---

## Output Format

When asked to package a skill end-to-end, produce, in this order:
1. The generalized skill name + one-line description
2. The one-page pitch (Section 3 format)
3. The three-tier ladder, filled in for this specific skill
4. Three scoping questions tailored to this skill
5. One outreach message draft

If the user is building a page (like a "skills marketplace"), format the pitch as a
card-ready block; if they're prepping a sales conversation, format the tiers and
scoping questions as talking points instead.
